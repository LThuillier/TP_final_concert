# Guide De Contribution

## Branches
- `main`: version stable.
- `develop`: integration continue.
- `feature/*`: nouvelles fonctionnalites.
- `bugfix/*`: corrections.

## Workflow Agile
1. Ouvrir une issue avec criteres d'acceptation.
2. Creer une branche `feature/*` ou `bugfix/*`.
3. Commits courts et explicites.
4. Ouvrir une Pull Request vers `develop`.
5. Validation par revue + tests automatiques.

## Definition Of Done
- Code relu.
- Tests passes.
- Aucune regression fonctionnelle.
- Documentation mise a jour (`docs/`).
